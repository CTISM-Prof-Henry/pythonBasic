Herana uma caracterstica central do paradigma orientado a objetos. a capacidade de certas classes herdarem caractersticas de outras

Vamos pensar em uma classe que descreve um objeto.

Quais caractersticas todos os objetos possuem?

Cdigo em Python:

In[25]:
class Objeto:
    def __init__(self, largura, altura, comprimento):
        """Construtor da classe Objeto"""
        self.largura = largura
        self.altura = altura
        self.comprimento = comprimento

caixa = Objeto(largura=10, altura=5, comprimento=3)

Vamos agora especializar esta classe, atravs de outra classe que ir herdar todos os atributos de Objeto, enquanto adiciona outros

  • Objeto:
    • Atributos: Altura, Largura, comprimento
    • Mtodos: nenhum
  • Animal:
    • Atributos: n patas, n olhos, peso
    • Mtodos: comunicar, respirar

Herdando a classe Objeto de maneira simples:

In[26]:
class Animal(Objeto):
    pass

A classe Animal possui todos os mtodos e atributos da classe Objeto, mas ainda no fez nenhuma especializao!

Precisamos sobrecarregar o construtor da classe Animal para aceitar mais parmetros, fazendo-o receber tambm os parmetros numero_patas, numero_olhos e cor

In[49]:
class Animal(Objeto):
    def __init__(self, largura, altura, comprimento, 
                 numero_patas, numero_olhos, peso):
        """Construtor da classe Animal"""
        super().__init__(largura, altura, comprimento)
        self.numero_patas = numero_patas
        self.numero_olhos = numero_olhos
        self.peso = peso


cachorro = Animal(
    largura=10, altura=30, comprimento=50, 
    numero_patas=4, numero_olhos=4, peso=15
)

O que esta linha de cdigo est fazendo?

super().__init__(largura, altura, comprimento, peso)

Ela chama o construtor __init__ da classe Objeto, que por sua vez superclasse (super) da classe Animal

Na prtica, como se a chamada ao mtodo super().__init__ copiasse e colasse o cdigo-fonte do construtor __init__ da classe Objeto:

In[28]:
class Animal(Objeto):
    def __init__(self, largura, altura, comprimento, 
                 numero_patas, numero_olhos, peso):
        """Construtor da classe Animal"""
        # cdigo copiado-e-colado do construtor de Objeto
        self.largura = largura
        self.altura = altura
        self.comprimento = comprimento
        # cdigo copiado-e-colado do construtor de Objeto
        
        self.numero_patas = numero_patas
        self.numero_olhos = numero_olhos
        self.peso = peso

Polimorfismo

possvel sobrescrever mais mtodos alm do construtor __init__. Vamos criar duas novas classes, Cachorro e Peixe. Cada uma destas classes ir sobrescrever a classe Animal:

In[30]:

    def comunicar(self):
        return 'Au! Au!'

    def respirar(self):
        return 'Arf! Arf!'
In[31]:
class Peixe(Animal):
    def comunicar(self):
        return 'Glub! Glub!'

    def respirar(self):
        return 'Glub! Glub!'

Outro detalhe: voc percebeu que no foi preciso reescrever o construtor de Animal nas classes Cachorro e Peixe?

In[51]:
class Cachorro(Animal):
    def comunicar(self):
        return 'Au! Au!'

    def respirar(self):
        return 'Arf! Arf!'

Quando no reescrevemos o construtor de Animal em Cachorro, no estamos sobrecarregando este mtodo. Desta forma, o comportamento do construtor em Cachorro ser o mesmo que o comportamento do construtor em Animal

Responda s seguintes perguntas:

possvel sobrescrever um mtodo em uma subclasse que muda a assinatura de parmetros da superclasse?

Sim!

J fizemos isso nessa aula:

class Objeto:
    def __init__(self, largura, altura, comprimento):
        """Construtor da classe Objeto"""
        self.largura = largura
        self.altura = altura
        self.comprimento = comprimento

class Animal(Objeto):
    def __init__(self, largura, altura, comprimento, 
                 numero_patas, numero_olhos, peso):
        """Construtor da classe Animal"""
        super().__init__(largura, altura, comprimento)
        self.numero_patas = numero_patas
        self.numero_olhos = numero_olhos
        self.peso = peso

possvel sobrescrever um mtodo em uma subclasse mais de uma vez?

No!

Outras linguagens de programao, como Java, permitem esse comportamento. Mas Python no:

In[56]:
class Vaca(Animal):
    # este mtodo sobrescreve comunicar de animal
    def comunicar(self):
        return 'Muuu!'

    # declarar um mtodo com o mesmo nome de outro mtodo
    # sobrescrito vai apagar o primeiro mtodo!
    def comunicar(self, parametro):
        return 'Muuu!'
